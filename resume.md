## Дополнительный опыт — Backend (Java / Spring)

Реализация REST API и сервисного слоя для нужд фронтенда.

### 1. Асинхронная обработка с прогрессом (IDM-2810) — 2025–2026, ~12 коммитов

- Реализация асинхронного выполнения операций с отслеживанием прогресса через REST API
- Архитектура `AsyncRestProcess` / `AsyncRestProcessManager` с сохранением контекста безопасности (`SecurityContextAwareSupplier`)
- DTO-модель прогресса (`ProgressDto`, `ProgressActivityDto`, `ProgressReportDto`)
- Интерфейс `ProgressReporter` для отчётов о выполняемых дельтах и деталях объектов ресурсов
- Очистка неиспользованных процессов, корректная сериализация `OperationResult`
- Оптимизация: замена `Callable` на `Supplier` в асинхронном исполнении

### 2. Смена пароля пользователем (IDM-2090) — 2024, ~14 коммитов

- REST-контроллер `CredentialsRestController` для изменения пароля
- Валидация пароля по политикам для каждого аккаунта с результатом валидации
- Шифрование пароля перед записью, обработка ошибок пропагации (partial_error)
- OpenAPI-документация для эндпоинтов
- Корректная обработка HTTP-статусов (отправка тела ответа даже при ошибках)

### 3. Экспорт объектов в CSV (IDM-3038) — 2025, ~10 коммитов

- REST-контроллер `ExportRestController` с сервисным слоем `ExportService`
- Система мапперов CSV-значений (`CsvValueMapper`, `CsvValueMapperFactory`, `FocusProjectionsMapper`, `DateTimeMapper`, `ObjectReferenceMapper`)
- Схема экспорта (XSD), документация схемы, тесты
- Корректное управление ресурсами (close resources)

### 4. Агрегированная статистика задач (IDM-659) — 2023, ~8 коммитов

- REST-контроллер `TaskStatisticsRestController` для агрегированной статистики
- Использование `ActivityStatisticsType` вместо кастомного класса
- Выделение логики формирования статистики в отдельный метод

### 5. Аудит и авторизация (IDM-1481, IDM-1365, IDM-406) — 2023–2024

- API-метод для получения слепка (реконструированного) объекта из события аудита (`AuditRestController`)
- Расширение роли End user для доступа в новый UI, `AuthController`
- Модификация конфигураций ролей (End user, Approver, Reviewer, Delegator)

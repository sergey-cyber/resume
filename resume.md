# Strong Middle+ Frontend-разработчик (React / TypeScript)

## Общий профиль

**Опыт:**
- 5 лет коммерческой разработки в АО СберТех
- Основная роль: Frontend-инженер (React/TypeScript), разработка и развитие enterprise-приложения (кодовая база ~70k строк)  
- Дополнительно: разработка backend на Java/Spring — разработка REST API, OpenAPI-спек

## Ключевые достижения

### 1. Спроектировал и реализовал систему авторизации на стороне UI с динамической ролевой моделью (~50+ ролей, настраиваемых из UI)
Система позволяет ограничивать видимость UI (страниц, компонентов, навигации, кнопок и т.п.) и действия пользователей. За все время эксплуатации — 0 инцидентов безопасности.
**Бизнес-ценность:** повышение безопасности, пользователь не может увидеть компонент/выполнить действие на которое у него нет прав.

### 2. Улучшил UX. Реализовал fullstack функционал, позволяющий выполненять тяжелые операции асинхронно
Проблема: в системе есть ряд тяжелых, занимающих большое количество времени, операций, которые выполнялись синхронно и блокировали UI до ее завершения (нажал кнопку, ждешь ~2 мин, наблюдая за спинером). Для решения этой проблемы доработал REST API, добавив возможность асинхронного выполнения операций в отдельном потоке с возможностью отслеживания прогресса и формирования отчета о выполнении операции, который можно детально рассмотреть на UI.
**Бизнес-ценность:** улучшен UX, теперь операция выпоняется в фоне, не блокируя UI. Пользователь может отслеживать ее выполнение в специальном прогресс баре или посмотреть результат позже, когда прийдет уведомление о завершении операции.

### 3. Миграция c Ant design на собственный UI KIT
Реализовал полую миграцию Ant Design компонентов на компоненты внутреннего UI KIT компании без потери качества. Были затронуты все экраны и компоненты приложения. Миграция проводилась поэтапно в течение ~6 месцев без регрессий и простоя приложения для пользователей. При этом парралельно занимался реализацией других задач.
**Ценность:** приведение интерфейса приложения к единому корпаративному стилю.

### 4. Реализовал систему построения сложных таблиц и фильтров/сортировок для них
В приложении очень много сущностей разного типа, которые отображаются в виде списков во всех ключевых разделах приложения. Списки имеют разный набор столбцов и большое количество фильтров/сортировок по разнличным параметрам. Параметров может быть очень много и разного типа.
Создал унифицированную schema-driven систему, которая позволет строить списки и связынные с ним фильтры динамически на базе схемы.
**Бизнес-ценность:** избавиляся от хардкода таблиц с фильтрами. Теперь добавлять, расширять и поддерживать списки проще и быстрее.

### 5. Реализовал fullstack функционал, позволющий экспортировать объекты системы
Реализовал rest-api, позволяющще экспортировать списки сущностей в виде csv файлов. Реализация включает в себя систему мапперов (mapper-factory) CSV-значений, что позваляет преобразовывать любые значения полей сущностей системы в строки и заполнть ими csv. На стороне UI предоставил удобный интерфейс, позволяющий софрмировать отфильтрованный, отсортированный список с нужным набором столбцов, который попадет в итоговый экспортируемый файл.
**Бизнес-ценность:** пользователи получили возможность самостоятельно выгружать данные для анализа в Excel/Google Sheets.

### 6. Качество кода и покрытие тестами
Обеспечил покрытие тестами не менее 80% кодовой базы. Для тестирования использовал E2E и компонентные тесты Cypress. Это позволило сократить количество регрессий при релизах и ускорить процесс рефакторинга — изменения в коде не ломают существующий функционал.
Поддерживал актуальность версий библиотек, использующихся в приложении.

### 7. Оптимизация производительности UI и HTTP-запросов
Провёл анализ и оптимизацию производительности приложения: внедрил виртуализацию для таблиц с 200+ строк, оптимизировал HTTP-запросы (кэширование, debounce, отмена дублирующихся запросов), что снизило нагрузку на сервер и ускорило отклик интерфейса.
**Бизнес-ценность:** улучшение UX — пользователи работают с приложением без задержек, снижение нагрузки на сервер.
